# Section Routing

Use this file to decide which VibeCode doc section to open first.

## Routing Rules

- Start with `quick-start` when the user wants a first app, deployment flow, or general orientation.
- Open `entity-api` for CRUD, search, aggregate, field schemas, auto-pagination, response formats, and entity conventions.
- Open `keys-auth` when the task involves key choice, scopes, OAuth, or why a request is unauthorized.
- Open `infra` for server creation, deploys, Black Hole, OPEN mode, ports, access policies, logs, exec, and lifecycle.
- Open `mcp` when the user wants AI agents, MCP transport, tool exposure, or IDE/agent integration.
- Open `cli` when the user wants `curl`, OpenAPI, `ocli`, or language snippets for direct API calls.
- Open `errors` when the user already has a failing request or wants retry/debug guidance.
- Open `api-reference` when a task needs endpoint-level confirmation after higher-level docs.
